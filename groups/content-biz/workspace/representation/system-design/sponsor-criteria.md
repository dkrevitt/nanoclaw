# Sponsor Criteria: System Design Newsletter

## Audience Context

Readers are backend/infrastructure engineers who care about:
- Scalability and reliability
- Distributed systems
- Production architecture
- System design interviews

## Pre-Filters

minFunding: Series A+ (need budget for $10k+ placements)
requireB2B: true
requireDevFocus: true

## must_have

- **Developer tool**: Product used by backend/infrastructure engineers
- **System relevance**: Can be explained in context of system design
- **Budget**: $10k+ for dedicated posts, $3k+ for banners
- **Real product**: Actually used in production by engineering teams

## nice_to_have

- Used by companies featured in System Design content (Stripe, Uber, etc.)
- Interesting architecture worth explaining
- Developer-loved (positive sentiment in community)
- Series potential (multiple angles to cover)

## exclude_if

- Frontend-only tools (not the audience)
- Consumer products
- No production use cases
- Requires heavy marketing spin

## Ideal Sponsor Categories

### Tier 1 (Perfect Fit)
- **Code review / collaboration**: CodeRabbit, Graphite, GitHub
- **Observability / monitoring**: Datadog, New Relic, Honeycomb
- **CI/CD / DevOps**: CircleCI, GitHub Actions, Buildkite
- **Databases**: MongoDB, CockroachDB, PlanetScale

### Tier 2 (Good Fit)
- **API tools**: Postman, Insomnia, Kong
- **Infrastructure**: Terraform, Pulumi, AWS/GCP/Azure
- **Developer productivity**: JetBrains, VS Code extensions
- **AI coding assistants**: Cursor, GitHub Copilot, Augment Code

### Tier 3 (Possible Fit)
- **Interview prep platforms**: If focused on system design
- **Learning platforms**: If technical/engineering focused
- **Career tools**: If developer-specific

## Sponsorship Packages

| Type | Price Range | Description |
|------|-------------|-------------|
| Dedicated post | $12-15k | Full newsletter on sponsor topic |
| Banner | $3-5k | Banner in regular newsletter |
| Series (4 posts) | $40-50k | Multiple dedicated posts |

## Content Integration

Best sponsorships feel natural within system design context:
- "How X Company Uses [Sponsor Product] to Handle Scale"
- "System Design Pattern: [Topic] (featuring [Sponsor])"
- "Engineering Best Practices: [Topic Relevant to Sponsor]"
